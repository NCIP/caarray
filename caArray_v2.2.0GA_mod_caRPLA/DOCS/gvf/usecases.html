Use Case Name

Upload caRPLA set of data files

Use Case ID

uploadSet

Primary Actor

user

Secondary Actor


Brief Description

Logged-in user can open to a signed applet that browses the local file system and displays .rpla-idf files in a selected directory. User selects an .rpla-idf file. This file gets parsed and added to file selection. There will be at least one .sradf file referenced in the .rpla-idf file, possibly more. Reference(s) to .sradf file(s) get read and the referenced file(s) get added to file selection. The caRPLA data set is validated and all distinct references to image files and array data files are resolved and included in file selection. User is presented with the file selection and confirms upload of all files into the caArray-caRPLA instance. Upload occurs via HTTP, avoiding firewall issues. Display of upload progress is shown to the user. After upload is complete, confirmation is displayed.


An experiment is created in the import of a caRPLA data set. All uploaded files are associated with this experiment.

Trigger


Pre-conditions

User has a caRPLA data set accessible from their local machine's filesystems.

Flow of Events


The exact mapping and workflow for processing uploaded caRPLA data files will be thoroughly defined in the Tech. Spec. section.


Post Conditions


Branch Point – <error handling>

Error if .sradf file, referenced from .rpla-idf file is not found.

Error if referenced array data files are not found.

Use Case Name

Upload additional data files

Use Case ID

uploadArrayDataFile

Primary Actor

user

Secondary Actor


Brief Description

Logged-in user can open to a signed applet page that browses the local file system and displays files in a selected directory.

User can upload a data file into the experiment. User must be able to associate the uploaded array data file with its appropriate source data file. The source data file could be either an image file, measured array data file, or derived array data file. For example, a measured array data file could be added to the experiment and get associated by the user to the appropriate image file.  The user could also add any number of derived array data files which may have either a measured array data file or other derived array data file as their sources. The user must be able to enter the required protocol used in the generation of the data file (or potentially browse, copy, and modify an existing protocol)

Use Case Name

Search for RPLArrayS through web UI

Use Case ID

searchUI

Primary Actor

user

Brief Description

Logged-in user is presented with a page to search RPLArrayS by at least the following criteria (to be refined)

experiment name
experiment identifier
RPLArray name
RPLArray identifier
array creator ( manufacturer)
specific protein target of Ab used
specific Ab used
Ab validation status
experiment
experimental factor category used
biosource id(s)
biosample id(s)
image filename
Boolean: whether RPLArray has associated raw imaging data, associated fitted data, or none of the above.
manufacturing protocol
manufacture date
substrate type
visibility

Upon completion of search, user is presented with sortable table of links (following caArray convention )to RPLArrays and the experiments to which they belong.

Selecting a link presents a page displaying the RPLArray characteristics and its experiment details. See viewRPLArrayAndExperiment use case for viewing details.
