Importing Data
Once files are uploaded into caArray, anyone associated with an experiment can validate and import annotation and array content files of the appropriate formats into the project. The import feature makes validated data available for download through the user interface and APIs.
While data is being imported into an experiment, all attributes and annotations of that experiment become read-only, so that user interface changes do not conflict with annotations being created as part of the import.
If multiple files with the same name (but different extensions) are imported, only one annotation chain of source > sample > extract > labeled extract > hybridization will be created, and all of the files will be associated with that single hybridization.
Required fields are turned off for data entry when the Service Type is Publish.
